    <ac:layout-cell>
      <p>Bem-vindos à nossa página dedicada ao Technical Improvement (Aprimoramento Técnico)! Aqui, exploraremos a importância fundamental de investir no aprimoramento contínuo de nossos componentes e produtos de Tecnologia da Informação. Em um mundo onde a inovação e a competitividade estão em constante evolução, é essencial que nosso time reconheça o valor intrínseco de buscar a excelência técnica.</p>
      <p>Em um cenário onde os avanços tecnológicos ocorrem em ritmo acelerado, a busca por melhorias técnicas é a chave para o sucesso sustentável de nossa organização. Nossos componentes e produtos de TI são a espinha dorsal de nossos serviços, e sua qualidade impacta diretamente a experiência de nossos clientes, a eficiência de nossos processos internos e, consequentemente, nosso crescimento no mercado.</p>
      <p>O Technical Improvement não se resume apenas a atualizar ferramentas e tecnologias, mas abrange uma abordagem holística para otimizar cada aspecto de nossos produtos e serviços. Engloba aperfeiçoamentos na arquitetura de software, design de interfaces, segurança, performance, escalabilidade e facilidade de manutenção, entre outros elementos cruciais para a robustez de nossas soluções.</p>
      <p>Além disso, o aprimoramento técnico é um fator determinante na redução de custos operacionais e na maximização do retorno sobre o investimento. Ao melhorar a eficiência e a confiabilidade de nossos componentes, podemos mitigar problemas recorrentes, minimizar retrabalho e diminuir a ocorrência de falhas, tornando nossa equipe mais produtiva e focada em iniciativas estratégicas.</p>
      <p>Nossos colaboradores também desempenham um papel essencial nesse processo. O incentivo à aprendizagem contínua e à troca de conhecimentos entre os membros da equipe não apenas fortalece nosso capital humano, mas também potencializa nossa capacidade de encontrar soluções inovadoras para desafios complexos.</p>
      <p>Nesta página, abordaremos os principais tópicos relacionados ao Technical Improvement, oferecendo recursos, melhores práticas, estudos de caso e oportunidades para o desenvolvimento do produto.</p>

      <p>
        <span>Os componentes, ou microserviços, em um projeto de software são unidades independentes e autônomas de funcionalidades que desempenham tarefas específicas dentro da aplicação como um todo. Cada microserviço é projetado para operar de maneira separada, com sua própria lógica de negócio e banco de dados dedicado, comunicando-se com outros componentes apenas por meio de interfaces bem definidas. Essa abordagem modular permite uma arquitetura flexível, escalável e de fácil manutenção, em que cada componente pode ser desenvolvido, implantado e atualizado independentemente, favorecendo a agilidade do desenvolvimento, a reutilização de código e a rápida resposta a mudanças de requisitos.             </em>
          </h2>
          <p>O Harbor é uma plataforma de registro de contêineres de código aberto desenvolvida pela VMware. Ele atua como um repositório seguro e confiável para armazenar, gerenciar e distribuir imagens de contêineres. O Harbor permite que equipes de desenvolvimento compartilhem, colaborem e implementem contêineres em ambientes de nuvem ou local com controle de acesso e políticas de segurança avançadas.</p>
          <p>Anteriormente, o nosso time compartilhava um repositório no Harbor com outras squads. Essa abordagem levou a problemas, como a necessidade constante de excluir imagens para liberar espaço e dificuldades no gerenciamento e aplicação de políticas remoção automática de imagens. Para resolver esses problemas, foi tomada a decisão de criar um repositório exclusivo para o projeto CRIM. Nesse novo repositório dedicado, o time pode gerenciar as imagens dos componentes do projeto de forma mais eficiente, com melhor controle sobre as políticas de acesso, sem se preocupar com o espaço ocupado por outras equipes. Essa mudança permitirá que o time tenha maior autonomia, organização e segurança na gestão de suas imagens de contêineres, garantindo um ambiente mais adequado e eficaz para o desenvolvimento do projeto CRIM.</p>

## ]]>🚧<![CDATA[ pré-requisitos
 
Para criar e executar uma Azure Function em um container Docker e no Kubernetes usando .NET 6, você precisa atender a alguns pré-requisitos. Aqui está uma lista dos principais requisitos:

1. SDK do .NET 6: Certifique-se de que você tenha o SDK do .NET 6 instalado no seu ambiente de desenvolvimento.

2. Docker: Tenha o Docker instalado e configurado na sua máquina local para criar e gerenciar containers.

3. Conta Azure: Você precisará de uma conta na plataforma Azure para criar e gerenciar recursos, como Azure Functions e Kubernetes Clusters.

4. Azure Functions Core Tools: Instale as Azure Functions Core Tools em sua máquina local para ajudar no desenvolvimento e implantação das funções.

5. Azure CLI: Instale a Azure Command-Line Interface (CLI) para interagir com recursos da Azure a partir do terminal.

6. Criação do Projeto: Crie um projeto .NET 6 Azure Functions usando o modelo de projeto correto. Por exemplo, usando o comando dotnet new func para criar um novo projeto Azure Functions.

7. Dockerfile: Crie um Dockerfile para empacotar a Azure Function dentro de um contêiner Docker. O Dockerfile é necessário para criar uma imagem do contêiner que será usada para a implantação.

8. Azure Container Registry (ACR): Crie um registro de contêiner (ACR) na sua conta Azure ou use outro registro de contêiner suportado para armazenar a imagem Docker da sua função.

9. Kubernetes Cluster: Configure e tenha acesso a um Kubernetes Cluster onde você planeja implantar as Azure Functions em contêineres.

10. Helm (opcional): Se você deseja usar o Helm para implantar recursos no Kubernetes, certifique-se de que o Helm esteja instalado no seu ambiente local.

11. Azure Functions Worker (versão adequada): Certifique-se de que a versão do Azure Functions Worker suporte o .NET 6. Verifique a documentação oficial da Microsoft Azure Functions para obter informações sobre a compatibilidade das versões.

12. Conhecimento básico de Docker e Kubernetes: É essencial ter conhecimento básico sobre Docker e Kubernetes, incluindo como criar, implantar e gerenciar contêineres e clusters.

Uma vez que você tenha atendido a esses pré-requisitos, poderá criar a Azure Function em .NET 6, construir uma imagem Docker, armazená-la no registro de contêiner e, em seguida, implantá-la em um cluster Kubernetes. Lembre-se de consultar a documentação oficial da Microsoft Azure para obter orientações detalhadas e atualizadas sobre como realizar cada etapa.

          </h2>
          <p>A Checkmarx é uma solução de Análise Estática de Segurança de Aplicações (SAST) originária de Israel, que opera na nuvem e tem como objetivo analisar o código-fonte das aplicações em busca de padrões relacionados a vulnerabilidades conhecidas. Além disso, o Checkmarx oferece uma funcionalidade de Análise de Componentes de Software (SCA) que permite melhorar a segurança em relação a bibliotecas de terceiros e questões legais relacionadas às licenças dessas bibliotecas.</p>
          <p>
            <ac:image ac:align="center" ac:width="400">
              <ri:url ri:value="https://swordfish-distribution.ru/wp-content/uploads/2021/07/sw-checkmarx-logo.png"/>
            </ac:image>
          </p>
          <h2 style="text-align: left;">
            <em>
              <strong>Configurando o Checkmarx na esteira:</strong>

          </h2>

          <h2 style="text-align: left;">
            <em>
              <strong>Como visualizar as vulnerabilidades encontradas pelo Checkmarx?</strong>
            </em>
          </h2>
          <p>A action do Checkmarx é integrada ao workflow, fornecendo um resumo da execução que inclui o número de vulnerabilidades encontradas e um link para acessar o Checkmarx e obter informações mais detalhadas (o acesso é gerenciado pela equipe de AppSec). Além disso, a action do Checkmarx disponibiliza o download de um relatório completo, contendo informações detalhadas sobre as vulnerabilidades identificadas. Dessa forma, os desenvolvedores podem ter uma visão geral das questões de segurança no próprio workflow e acessar os dados mais completos conforme necessário para realizar a correção das vulnerabilidades.</p>
          <p>Atualmente a esteira não está sendo bloqueada em função das vulnerabilidades encontradas, porém no futuro o time de Esteira Contínua habilitará uma trava quando existirem pendências do Checkmarx.</p>
